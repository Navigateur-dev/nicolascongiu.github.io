Projet PHP : Gestion des remboursements de frais

I. Résumé de la mission :

Dans un premier temps, l'objectif de ce projet était de corriger des bugs relévés par des utilisateurs (voir document 1). Dans un second temps, des fonctionnalités d'évolution de l'application étaient à réaliser (voir document 2, 4 et 5), chaque membre de mon équipe devions implémenter des fonctionnalités CRUD (Create, Read, Update, Delete) pour chacune des gestions à réaliser. Pour ma part, je devais gérer le CRUD des déplacements de délégué à une pharmacie pour la présentation d'un produit à un médecin. La convention PSR était à respecter (voir document 4). Afin d'avoir une meilleure organisation, l'utilisation de Git, Github (document 3) pour le gérer les versions des fonctionnalités et Trello (voir document 6) pour le suivi du projet étaient les outils à utiliser.

-Développer la présence en ligne de l'organisation :
->Participer à l'évolution d'un site Web exploitant les données de l'organisation.
-Travailler en mode projet :
->Analyser les objectifs et les modalités d'organisation d'un projet;
->Planifier les activités;
->Évaluer les indicateurs de suivi d’un projet et analyser les écarts;

III. Environnement et outils :

Projet Dotnet : Gestion des Entrepôts

I. Résumé de la mission :

Au cours de cet AP, une application client lourd en C# était à développer pour aider une entreprise à gérer le contrôle des stockes de produits dans des zones de stockages par des entreprises. En mode projet par groupe de 4, il était nécessaire d'utiliser Trello (voir Document 1) afin d'organiser les missions et Git et github (voir Document 2) pour gérer les versions des fonctionnalités. Des réunions ont été mise en place chaque semaine pour un suivi de l'avancé du projet de la même manière que la méthode AGILE (en sprint)(voir Document 3). Un plan de test pour contrôler les fonctionnalités était à préparer (voir Document 4) et enfin une documentation utilisateur pour les fonctionnalités pour faciliter l'utilisateur dans l'usage de la solution. (voir Document 5). Enfin, l'application devait comporter une gestion des fonctionnalités pour attribuer les droits aux utilisateurs dans chacunes des fonctionnalités. (voir Document 6)
